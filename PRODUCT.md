# Product

## Register

product

## Users

The primary user is a technical operator running Media2Text as a local-first
headless ingestion service. The operator uses the web UI to configure sources,
start or inspect runs, monitor health, audit generated artifacts, and understand
what still needs backend work.

## Product Purpose

Media2Text turns YouTube videos, channels, playlists, and direct audio inputs
into structured transcript artifacts for downstream automation and AI services.
The UI is an operator console for configuration, monitoring, and audit. It is
not a marketing page and it is not the final daily reading or analysis surface.

## Brand Personality

Precise, technical, honest.

## Anti-references

Avoid SaaS marketing dashboards, decorative gradients, glass cards, oversized
hero sections, rounded consumer-app styling, and any UI that presents roadmap
data as if it were live backend truth.

## Design Principles

- Keep the engine headless: UI supports the service, API, scheduler, and files.
- Show real backend state where available; mark partial, estimated, and planned
  functionality explicitly.
- Preserve the Media2Text visible brand while keeping the `youtube2text` and
  `Y2T_` technical contract intact.
- Prioritize dense operator workflows over promotional storytelling.
- Make transcript artifacts easy to inspect, copy, and pass to downstream tools.

## Accessibility & Inclusion

Target WCAG AA contrast, keyboard-accessible controls, visible focus states,
44px mobile touch targets, readable technical labels, and reduced-motion-safe
transitions.
